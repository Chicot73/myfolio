Характерные принципы работы:

Создаю дизайн с учетом пожеланий заказчика, согласовывая все изменения. Благодаря работающей системе поиска уникальных идей успешно решаю с заказчиком стоящие перед его компанией маркетинговые задачи: например, привлечение новых клиентов, раскрутка новых позиций в линейке товаров / услуг и т.д.

Верстаю готовые работающие сайты на основе дизайн-макетов. Работал с макетами PSD, Adobe XD и Figma. Знаю, как правильно экспортировать графику из макета, нахожу и подключаю все необходимые шрифты, узнаю точные отступы. Если есть возможность достать из макета SVG, использую его а не растровый вариант.

Создаю макет сайта с учетом его адаптива на всех устройствах, если это есть в брифе.

«Пиксель-в-пиксель»

Понимаю, как важно сделать верстку точно по макету, поэтому верстаю по принципу «пиксель-в-пиксель», если потребуется. В процессе работы накладываю дизайн-макет на верстку, чтобы убедиться, что расхождений нет.

Семантическая верстка

Делаю семантически правильную HTML-разметку, без проблем прохожу проверку кода на валидность без ошибок. Понимаю, как расставить теги и заголовки на странице, чтобы SEO-специалисту не нужно было корректировать мой код.

Адаптивная верстка

Делаю адаптив сайта не с помощью библиотеки Bootstrap, а с помощью тега <media>, что не привязывает к определенной технологии и работает на всех OC.

Стили на препроцессорах

CSS-стили пишу с помощью препроцессора SASS, поэтому получается верстать быстро, легко находить нужный участок кода и делать гибкие настройки. Например, если стиль повторяется два раза — пишу миксин и дальше использую уже его вместо копи-паста.

Хостинг, домен, Git и т.д.

Умею настраивать хостинг, привязывать домен и регистрировать SSL-сертификат. Сам проект обычно храню в репозитории на Github. Хотя еще не участвовал в командной разработке, поэтому Git знаю на уровне: сделать репозиторий, клонировать его, внести изменения, сохранить коммиты и отправить пуш.
